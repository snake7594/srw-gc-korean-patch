# 슈퍼로봇대전 GC 한글패치

닌텐도 게임큐브 일본판 **슈퍼로봇대전 GC**용 비공식 한국어 패치입니다. 대상 게임 ID는 `GRWJD9`이며, 다른 지역판이나 이미 수정된 이미지에는 적용할 수 없습니다.

최신 배포본은 [v1.0.10 릴리스](https://github.com/snake7594/srw-gc-korean-patch/releases/tag/v1.0.10)에서 받을 수 있습니다.

**v1.0.1은 전투 화면의 동적 타일 번호를 손상할 수 있으므로 사용하지 마세요.**
**v1.0.3은 인물·로봇도감 설명문에서 글자 깨짐과 잘못된 메모리 읽기가 발생할 수 있어 철회했으므로 사용하지 마세요.**
**v1.0.4는 정신기 설명 23건이 다른 정신기에 연결되므로 v1.0.10으로 교체하세요.**

## v1.0.10 주요 내용

메뉴 라벨이 옆 칸을 침범하던 문제를 고치고, 타이틀 로고를 일본판 원본으로 되돌린 릴리스입니다. 번역 텍스트·글꼴은 v1.0.9와 같고 `add00dat.bin`만 달라집니다.

- 일본판과 캔버스 치수가 다르던 번역 라벨 **23건**(능력 창의 레벨·원호·잔여·합계·지형 4열 등)을 일본판 치수로 복원 — 게임 레이아웃보다 넓어 인접 수치 칸을 덮던 11건의 침범 해소
- 이로써 번역 라벨 452개 전부가 일본판 retail과 캔버스 치수 일치 (불일치 0건)
- 타이틀 화면의 한글 로고를 **일본판 원본 로고(スーパーロボット大戦)**로 복원 (블록 10개 바이트 동일)
- 라벨 51건에 일본판식 선반 장식을 재구성해 원본과 같은 외형으로 통일
- 시나리오 제목 이미지·도감·로딩 화면 등 다른 한글 그래픽은 유지

## v1.0.9 주요 내용

메뉴 이미지의 한글을 일본어 원문과 같은 길이·밀도로 다듬은 릴리스입니다. 대사·메뉴 번역 텍스트와 글꼴은 v1.0.8과 같고 `add00dat.bin`(UI 이미지)만 달라집니다.

- 일본판 원문보다 길어 작게 그려지던 라벨 **104개**를 원문 길이에 맞게 정리
  - 원문이 한자 1자인 지형 표기: 공중→**공**, 지상→**육**, 수중→**해**, 우주→**우**
  - 원문이 영문인 라벨은 영문 유지: 체력→HP, 에너지→EN, 정신→SP, 브이맥스→V-MAX 등 13건
  - 원문에 없는 띄어쓰기 제거(강화 파츠→강화파츠 등)와 축약(정신기→정신, のりかえ→환승) — 번역 변경 74건, 전건 일본판 원문과 대조 확인
- 일본판 원본 UI 글꼴과 같은 **가로 압축 렌더링** 도입(자당 최소 0.6 비율) — 원문과 같은 글자 수로도 폭이 넘치던 라벨을 원본과 같은 밀도로 표기
- 좁은 캔버스 추가 복원 15건(총 152건)
- 결과: 대상 104개의 일본판 대비 잉크 높이 평균 0.803 → **0.969**, 최저 0.435 → **0.909** (잔여 차이는 전부 1px 수준)
- 독립 검증: 별도 에이전트가 자체 렌더러로 재측정, 압축 최강 라벨의 가독성 육안 검수 통과

## v1.0.8 주요 내용

글꼴을 나눔스퀘어 네오로 바꾸고, 메뉴 이미지 크기를 일본판 원본에 맞춘 릴리스입니다. 대사·메뉴 번역 내용은 v1.0.7과 같습니다.

- 대사 글꼴과 메뉴 이미지 글꼴을 **나눔스퀘어 네오 Bold(NanumSquareNeo-cBd)**로 교체
  - 런타임 한글 글리프 1,370자를 전부 다시 렌더링 (문자 코드 배치는 그대로 유지)
- 메뉴 이미지의 한글 크기를 **일본판 원본 잉크 높이·위치 기준**으로 재조정
  - 기존에는 영문판 레이아웃에 맞춰져 일본판보다 크거나(33건) 작게(1/3 크기 포함) 그려짐
  - 메뉴 상자(SCR 캔버스) 치수가 영문판 유래로 일본판보다 좁던 137개를 일본판 치수로 복원
  - 일본판 높이 ±5% 일치 라벨 247 → 283건, 심각한 과소(0.6배 미만) 21 → 10건, 과대 렌더링 0건
- 전투 결과 라벨(명중·치명타 등 23종)은 타일 번호 한계(여유 64개, 필요 글자 58자) 때문에 8×8 축소 표기를 유지
- `리알로봇계` 표기를 `리얼로봇계`로 교정
- 원본 아틀라스 타일 번호·동적 참조 후보 타일 전부 보존, 공유 아틀라스 감사 통과

## v1.0.7 주요 내용

대사가 대사창 밖으로 넘쳐 잘리던 문제를 고친 릴리스입니다. 번역 내용은 v1.0.6과 같고 줄바꿈 위치만 달라진 대사가 대부분입니다.

- 대사창 규격을 실기에서 확인: **한 줄 18자(「와 」 포함), 한 화면 3줄**. 일본판 원문도 이 규격을 지킵니다(add01 블록 883개 중 866개가 모든 줄 18자 이하)
- 규격을 넘던 대사 **4,252건을 정렬**
  - 시나리오 대사 576건, 전투 대사 3,676건
  - 4,024건은 번역 문구를 그대로 두고 줄바꿈만 이동, 228건은 문장이 길어 뜻을 유지한 채 압축
- 전투 대사에서 일본판이 여러 줄로 나눈 대사가 한 줄로 합쳐져 있던 문제 수정. 가장 긴 줄은 82자로 창을 크게 벗어났습니다(v1.0.0부터 있던 문제이며 기존 번역 데이터에서 비롯됨)
- 같은 대사가 두 번 반복되거나 후보 번역 두 개가 이어붙어 있던 전투 대사 정리
- 프롤로그 내레이션(화자 없는 도입부)은 원문이 25자까지 쓰는 넓은 창이므로 그 폭까지 허용
- 대사 57,697건 전수 검사에서 창을 넘는 줄 **0건** 확인. 이 검사를 `tools/audit_window_fit.py`로 추가해 재발을 막습니다

## 적용 방법

1. [xdelta3 공식 프로젝트](https://github.com/jmacd/xdelta-gpl/releases)에서 Windows용 `xdelta3.exe`를 준비합니다.
2. 다음 파일을 같은 폴더에 둡니다.
   - `SRW_GC_Korean_v1.0.10.xdelta`
   - `APPLY_PATCH.bat`
   - `apply_patch.ps1`
   - `xdelta3.exe` — 또는 `xdelta3`/`xdelta`를 `PATH`에 등록
3. 명령 프롬프트에서 아래처럼 실행합니다.

```bat
APPLY_PATCH.bat "D:\Games\Super Robot Taisen GC.iso"
```

출력 경로를 직접 정하려면 두 번째 인수를 지정합니다.

```bat
APPLY_PATCH.bat "D:\Games\Super Robot Taisen GC.iso" "D:\Games\Super Robot Taisen GC Korean.iso"
```

출력 경로를 생략하면 원본 ISO와 같은 폴더에 `Super Robot Taisen GC_Korean_v1.0.10.iso`가 만들어집니다. 스크립트는 패치 전 원본과 패치 후 결과의 SHA-256을 모두 검사합니다.

PowerShell에서 직접 실행할 수도 있습니다.

```powershell
.\apply_patch.ps1 -SourceIso 'D:\Games\Super Robot Taisen GC.iso'
```

## 저장소 구성

- `tools/`: 일본판 데이터 재구성, 문자열 재배치 및 독립 감사를 위한 소스 코드
- `tools/repack_preserve_indices.py`: 원본 타일 번호를 보존하는 최종 UI 재패커
- `tools/audit_shared_atlas_refs.py`: 동적·공유 타일 손상 여부 감사 도구
- `tools/apply_translation_quality_overrides.py`: PDF·일본어 원문 교정값을 안정 ID 기준으로 적용하고 구조·문자 잔존을 검사하는 도구
- `tools/apply_spirit_command_corrections.py`: 정신기 이름·축약·설명을 일본판 안정 ID로 교정하고 30·30·29건의 대응을 검사하는 도구
- `tools/patch_episode_title_graphics.py`: 허용된 시나리오 제목 BMP만 고정 레이아웃으로 다시 그리는 도구
- `tools/build_iso_from_maps.py`: 번역 맵에서 바이너리 조립과 ISO 재배치를 한 번에 수행하고, `--expect-sha`로 기존 릴리스의 재현을 검증하는 도구
- `tools/extract_mt_targets.py`, `tools/chunk_targets.py`: 기계번역 잔여 텍스트를 문맥·용어집과 함께 추출하고 검수 단위로 분할하는 도구
- `tools/validate_mt_output.py`: 교정본을 실제 빌드 인코더로 검증(표시 폭·구조 토큰·글꼴 코드북·화자 보존)하는 도구
- `tools/add01_payload.py`, `tools/lib_payload.py`: 대사의 화자 구간과 도감 설명 줄바꿈을 빌드와 동일한 규칙으로 처리하는 모듈
- `tools/audit_speaker_prefix.py`: 대사 교정이 화자 이름 구간을 덮어쓰지 않았는지 감사하는 도구
- `docs/기술_문서.md`: 패치 구조와 검증 절차
- [번역 JSON 수정 가이드](docs/번역_JSON_수정_가이드.md): 대사·메뉴·도감·정신기 번역을 수정할 JSON과 안정 ID 작성 방법
- [번역 문체 가이드](docs/번역_문체_가이드.md): 대사 교정 시의 말투·표기·구조 토큰 규칙
- `tools/dialogue_fit.py`, `tools/bpilot_fit.py`: 대사를 대사창 규격(18자·3줄)에 맞게 줄바꿈을 다시 계산하는 도구
- `tools/audit_window_fit.py`: 대사 전량이 대사창을 넘지 않는지 검사하는 도구
- `tools/ui_text_fit.py`: 메뉴 한글을 일본판 라벨의 잉크 높이·세로 중심에 맞춰 배치하는 공용 모듈
- `tools/restore_jp_scr_canvas.py`: 영문판 유래로 좁아진 메뉴 상자(SCR 캔버스)를 일본판 치수로 복원하는 도구
- `tools/rerender_codebook_font.py`: 문자 코드 배치를 유지한 채 런타임 한글 글리프만 새 글꼴로 다시 그리는 도구
- `docs/UI_이미지_패치.md`: 메뉴·타이틀·로딩 이미지의 재패킹 방식과 재현값
- `data/`: UI 이미지 매핑, PDF 번역 품질 교정 및 정신기 안정 ID 교정 JSON
- `APPLY_PATCH.bat`, `apply_patch.ps1`: Windows용 패치 적용 도구
- `release_manifest.json`, `SHA256SUMS.txt`: 릴리스 식별값과 파일 검증값

게임 ROM, 추출한 게임 바이너리, 원문 전체 덤프, 전체 대사 데이터, 참조 PDF 및 xdelta 실행 파일은 이 저장소에 포함하지 않습니다. 참조 PDF는 파일명·페이지·SHA-256만 교정 근거로 기록합니다.

## 문제 해결

- **원본 해시가 다름**: 다른 지역판, 압축 해제 과정에서 변형된 이미지 또는 기존 패치가 적용된 이미지일 수 있습니다.
- **xdelta3를 찾지 못함**: `xdelta3.exe`를 적용 스크립트와 같은 폴더에 두거나 실행 파일을 `PATH`에 등록하세요.
- **출력 파일이 이미 있음**: 데이터 손상을 막기 위해 자동으로 덮어쓰지 않습니다. 기존 파일을 다른 곳으로 옮기거나 다른 출력 경로를 지정하세요.

## 법적 고지

이 프로젝트는 원저작권자 및 관련 회사와 관계없는 팬 번역 프로젝트입니다. 패치에는 원본 게임 ISO가 포함되지 않으며, 원본 게임에 관한 모든 권리는 각 권리자에게 있습니다. 패치 사용자는 자신이 합법적으로 소유한 정품에서 원본 이미지를 직접 준비해야 합니다. ROM 이미지의 공유·판매·재배포를 금합니다.
