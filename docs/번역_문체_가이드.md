# 슈퍼로봇대전 GC 한글패치 — 기계번역 교정 스타일 가이드

## 목표
일본어 원문(display 형태, `\n` = 줄바꿈)을 보고 현재 기계번역 한국어를 **자연스러운 한국어 게임 대사**로 고친다.
이미 충분히 자연스럽고 정확하면 고치지 않는다(`"keep": true`).

## 절대 규칙 (검증기가 강제)
1. **줄 수 유지**: 출력 줄 수는 일본어 display 줄 수와 정확히 같아야 한다 (add01). bpilot은 원문 이하 줄 수 허용.
2. **구조 토큰 보존**: `<AA>`, `<FF>`, `<TT>` 토큰은 개수·형태 그대로 유지. `<AA>`는 런타임에 파일럿 이름 등으로 치환되는 동적 토큰이다. 절대 삭제·번역·추가 금지.
3. **일본어 문자 금지**: 히라가나·가타카나·한자가 남으면 안 된다. 단 `・`(가운뎃점)는 한국어 표기 관례로 허용.
4. **글자 제한**: 한글은 코드북에 있는 1,346자만 사용 가능. 평범한 한국어 표현은 대부분 통과하니 일부러 특이한 음절(예: 뷁, 챣)을 만들지 말 것. 검증기에서 거부되면 다른 표현으로 재작성.
5. **줄 폭**: add01 대사 한 줄 ≤ 19자(전각 기준), bpilot ≤ 22자, add02/dol은 현재 길이 수준 유지(±20%).

## 문체 규칙
- **말투는 일본어 원문의 어조를 따른다**: 반말(だ・である·くだけた)은 반말로, 정중어(です・ます)는 해요체/합쇼체로. 군인 상관에게는 "~합니다/~입니다", 동료·친구끼리는 반말.
- 어미 처리: 「～だぜ」「～だな」 같은 남성적 어미 → "~라고", "~군" 등 자연스럽게. 「～わ」「～のよ」 여성 어미 → 부드러운 종결어미.
- **캐릭터 이름·용어는 용어집(glossary.json)을 따른다**. speakers 맵의 한국어 표기가 정본. 용어집에 없으면 통용 한국어 표기(원작 애니 국내 표기)를 쓴다.
- 주인공 이름: 남주인공 = 아카츠키 아키미, 여주인공 = 아카츠키 아케미.
- 로봇/기체명은 r_name 용어집 표기를 따른다 (예: 마징가Ｚ, 겟타로보).

## 표기 관례 (기존 인간 번역과 통일)
- 대사 인용 괄호 「」 『』 는 원문 그대로 유지.
- 말줄임표: 원문의 `・・・`/`…` → `…` (U+2026) 사용. `...`(마침표 3개) 금지.
- 쉼표·마침표: 전각 `、` `。` 를 원문 위치에 맞게 유지하되, 한국어 어순에 맞게 재배치 가능. 순수 한국어 쉼표 `,` 도 허용(기존 번역에서 혼용).
- 느낌표/물음표: 전각 `！` `？` 권장 (반각도 자동 변환되므로 허용).
- 숫자·로마자: 전각(０１２, ＡＢＣ) 유지 권장. ＨＰ, ＢＯＤＹ 같은 게임 용어는 그대로.
- 공백: 전각공백(　)은 레이아웃용이므로 필요시 유지, 불필요하면 제거 가능.

## 품질 기준 (무엇을 고치나)
기계번역의 전형적 문제:
- 조사 오류: "지구은" → "지구는"
- 존댓말/반말 뒤섞임: 한 대사 안에서 "~습니다" 와 "~야" 혼용 → 캐릭터에 맞게 통일
- 직역투: "~하는 것이 가능하다" → "~할 수 있다"
- 미번역 음차: "이프크" 같은 의미 없는 음차 → 의미 번역
- 어색한 띄어쓰기·문장부호: "고생이었지?페어리" → "고생했네? 페어리"
- 문맥 무시: 전후 대사(context)를 보고 지시어·호칭·어조를 맞춘다

## 출력 형식
각 레코드에 대해 JSON 객체:
- 고칠 때: `{"id": "<그대로>", "lines": ["1행", "2행", ...]}` — lines는 **대사 본문만** (화자 이름 제외), 줄 수 규칙 준수
- 유지할 때: `{"id": "<그대로>", "keep": true}`
