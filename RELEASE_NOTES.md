# 슈퍼로봇대전 GC 한글패치 v1.0.8

글꼴을 나눔스퀘어 네오로 바꾸고, 메뉴 이미지의 한글 크기를 일본판 원본에 맞춘 릴리스입니다. 대사·메뉴의 번역 내용은 v1.0.7과 같으며, 대사를 담은 `add01dat.bin`과 `bpilot.pak`, 메뉴 텍스트 `add02dat.bin`은 v1.0.7과 SHA-256이 동일합니다.

## 글꼴 교체

- 대사 글꼴과 메뉴 이미지 글꼴을 **나눔스퀘어 네오 Bold(NanumSquareNeo-cBd)**로 교체했습니다.
- 런타임 한글 글리프 1,370자를 전부 다시 렌더링했습니다. 문자 코드 배치는 바이트 단위로 그대로 유지되므로 기존 텍스트 데이터와의 대응이 바뀌지 않습니다.
- 18×18 글리프 상자 안에서 잘림 없이 들어가는 최대 크기(17px)를 1,370자 전수 검사로 선정했고, 획 뭉갬(ㅇ·ㅁ·ㅂ 속 구멍 보존)도 기존 글꼴과 동급 이상임을 확인했습니다.

## 메뉴 이미지 크기 정합

기존 메뉴 이미지는 영문판 레이아웃을 기준으로 그려져 일본판 원본과 크기가 맞지 않았습니다.

- 원인 1 — 렌더링 기준: 한글을 영문판 라벨 상자에 맞춰 축소하던 것을, **일본판 원본 라벨의 잉크 높이·세로 중심에 맞추도록** 변경했습니다.
- 원인 2 — 상자 치수: 메뉴 상자(SCR 캔버스) 자체가 영문판 유래로 일본판보다 좁은 라벨이 173개 있었고, 이 중 137개를 **일본판 retail 치수로 복원**했습니다. 일본판 ISO를 패치하므로 일본판 치수가 게임이 기대하는 값입니다. (일본판 상자가 오히려 좁아 한글이 작아지는 20개 등은 현행 유지)
- 결과: 일본판 높이 ±5% 일치 라벨 247 → 283건, 심각한 과소(0.6배 미만) 21 → 10건, 일본판보다 크게 그려지던 라벨 33 → 0건.
- `리알로봇계` 표기를 `리얼로봇계`로 교정했습니다.

## 유지되는 제한

- 전투 결과 라벨 23종(명중·치명타·합체 공격 등)은 해당 아틀라스의 타일 번호 여유가 64개뿐이고 필요한 고유 글자가 58자라 8×8 축소 표기를 유지합니다. 키우려면 타일이 116개 이상 필요해 전투 그래픽을 침범합니다.
- 남은 과소 10건은 일본판 상자 자체가 좁거나(한자 1자 라벨 등) 한글 글자 수가 많은 경우입니다.

## 검증

- 복원한 SCR 137개는 헤더와 블록 길이가 일본판 retail과 바이트 단위로 일치하며, retail과 다른 바이트는 한글 타일을 가리키는 타일 번호뿐입니다.
- 원본 아틀라스 타일 변경 0건, 보존 대상 SCR 재작성 0건, 동적 참조 후보 타일 1,383개 보존 — 공유 아틀라스 감사 통과(종료 코드 0).
- 독립 검증 에이전트가 별도 작성한 파서·렌더러로 산출물을 재측정해 위 수치가 전부 재현됨을 확인했습니다.
- 빌드 산출물 감사: 텍스트 페이로드 62,506건 검증 실패 0건, 일본어 잔존 0건, 승인 외 실행 코드 변경 0건, 일본판 비대상 파일 277개 동일.
- 대사창 규격(18자·3줄) 전수 검사 57,697건 위반 0건 유지.
- xdelta 역적용 SHA-256 일치, 배포 `apply_patch.ps1` 실제 실행 검증.

| 원본 SHA-256 | `AD4CB99FFB3C0383802A2AB87963F98BA417DFC5184ED3FE3DFE077DA02DB229` |
| 결과 SHA-256 | `645A30E3DAA781F622966BA940164A82FED52F0A65FABAB476CC60A0845BFF4F` |
| xdelta 크기 | `109,511,589` 바이트 |

수정되지 않은 일본판 원본 ISO에만 적용하세요. 이전 버전이 적용된 이미지 위에 덧씌우면 안 됩니다.

## 이전 버전 주의

- **v1.0.1**은 전투 화면의 동적 타일 번호를 손상할 수 있으므로 사용하지 마세요.
- **v1.0.3**은 인물·로봇도감에서 글자 깨짐과 잘못된 메모리 읽기가 발생해 철회했습니다.
- **v1.0.4**는 정신기 설명 23건이 다른 정신기에 연결되므로 v1.0.8로 교체하세요.
- **v1.0.6**은 대사 4,318곳이 대사창을 넘으므로 v1.0.8로 교체하세요.

## 참고

- 나눔스퀘어 네오 글꼴은 네이버가 배포하는 무료 글꼴이며, 글꼴 파일 자체는 저장소와 릴리스에 포함하지 않습니다. 빌드 재현 시 `NanumSquareNeo-cBd.ttf`를 직접 준비해야 합니다(고정 SHA-256 검사 있음).
- 자동 감사와 표본 검수로 검증했으며, 에뮬레이터·실기에서의 전체 플레이 검증은 하지 않았습니다.

# 슈퍼로봇대전 GC 한글패치 v1.0.7

대사가 대사창 밖으로 넘쳐 잘리던 문제를 고친 릴리스입니다. 번역 내용은 v1.0.6과 같고, 대부분은 문구를 그대로 둔 채 줄바꿈 위치만 달라졌습니다.

## 대사창 규격

실기 확인 결과 대사창은 **한 줄에 「와 」를 포함해 18자**, **한 화면에 3줄**까지 표시합니다. 일본판 원문도 이 규격을 지킵니다. add01 블록 883개 중 866개가 모든 줄을 18자 이하로 유지하고, 전투 대사의 가장 긴 원문 줄도 19자입니다.

예외도 원문이 증명하는 범위만 허용했습니다.

- 화자 없는 프롤로그 내레이션(블록 0)은 원문이 25자까지 사용하는 넓은 창입니다.
- 원문에 19자 줄이 하나씩 있는 블록 16개는 그 폭까지 허용합니다.

## 수정 내용

- 규격을 넘던 대사 **4,252건 정렬** — 시나리오 대사 576건, 전투 대사 3,676건
  - **4,024건은 번역 문구를 한 글자도 바꾸지 않고** 줄바꿈만 다시 계산했습니다.
  - 228건은 문장이 길어 한 줄에 담기지 않아, 뜻과 말투를 유지한 채 압축했습니다.
- 전투 대사에서 일본판이 여러 줄로 나눈 대사가 한 줄로 합쳐져 있던 문제를 수정했습니다. 가장 긴 줄은 82자로 창을 크게 벗어나 있었습니다. v1.0.0부터 있던 문제이며 기존 번역 데이터에서 비롯된 것으로, 초과 3,670건 중 3,637건이 여기에 해당합니다.
- 같은 대사가 두 번 반복되거나 후보 번역 두 개가 이어붙어 있던 전투 대사를 정리했습니다.
- 전투 대사에 빈 줄이 섞여 화면 줄 수를 넘기던 레코드를 정리했습니다.

## 검증

- 시나리오 대사 22,378건과 전투 대사 35,319건, 총 **57,697건 전수 검사에서 창을 넘는 줄 0건, 줄 수 초과 0건**을 확인했습니다. v1.0.6은 같은 검사에서 4,318건이 걸립니다.
- 줄바꿈만 옮긴 4,024건은 공백을 제외한 문자열이 수정 전과 완전히 같음을 대조해, 문구가 바뀌지 않았음을 확인했습니다.
- 압축한 228건은 실제 빌드 인코더로 검증했습니다. 표시 폭·구조 토큰·글꼴 코드북·일본어 잔존 위반 0건입니다.
- 빌드 산출물 독립 감사에서 텍스트 페이로드 62,506건 검증 실패 0건, 승인 외 실행 코드 변경 0건, 일본판 비대상 파일 277개 동일을 확인했습니다.
- xdelta 패치를 원본에 역적용해 결과 ISO의 SHA-256 일치를 확인하고, 배포되는 `apply_patch.ps1`을 실제로 실행해 재현을 검증했습니다.

이 검사는 `tools/audit_window_fit.py`로 저장소에 추가했으며, 앞으로 같은 문제가 다시 실리지 않도록 빌드 전 검사에 사용합니다.

## 변경 범위

메뉴·도감·시스템 문구와 그래픽·글꼴·오프닝은 v1.0.6과 동일합니다. `add02dat.bin`, `add00dat.bin`, `font.pak`, `opening.bnr`, `Start.dol`은 v1.0.6과 SHA-256이 같고, 대사를 담은 `add01dat.bin`과 `bpilot.pak`만 달라집니다.

# 슈퍼로봇대전 GC 한글패치 v1.0.6

v1.0.5까지 기계번역 상태로 남아 있던 문장 19,614건을 일본어 원문과 전후 문맥에 대조해 다시 번역한 릴리스입니다. 게임 구조, UI 이미지, 글꼴, 정신기 교정, 도감 런타임 수정은 v1.0.5와 동일합니다. 실제로 `add00dat.bin`, `font.pak`, `opening.bnr`은 v1.0.5와 SHA-256이 같으며, 대사·메뉴·전투·시스템 텍스트를 담은 파일만 달라집니다.

## 번역 품질 개선

- 기계번역으로 남아 있던 텍스트 **19,614건**을 일본어 원문·화자·전후 문맥 기준으로 재번역
  - 시나리오 대사 12,079건
  - 전투 대사 6,911건
  - 메뉴·도감 495건
  - 실행 파일 내 시스템 문구 129건
- 의미가 뒤바뀌어 있던 오역 교정. 예를 들어 `実の息子`가 `열매의 아들`로, 노인의 1인칭 `ワシ`가 `독수리`로, `好きにしろ`가 `좋아해!`로 옮겨져 있던 문장을 각각 `친아들`, `나`, `마음대로 해라`로 수정
- 한 대사 안에서 뒤섞이던 존댓말과 반말을 화자와 관계에 맞게 통일하고, 조사 오류(`지구은` → `지구는`)와 직역투 정리
- 인물·로봇도감 설명 482건을 재번역. 예를 들어 `직정형의 시노부란충돌을 반복하고`처럼 문장이 무너져 있던 설명을 `직정적인 시노부와는 충돌을 거듭했다`로 교정
- 의미 없이 음차되어 있던 표현을 뜻이 통하는 한국어로 교정
- 인명·기체명·용어는 기존 확정 번역에서 만든 용어집에 맞춰 통일

## 검증

- 교정본 전량을 실제 빌드 인코더로 검증했습니다. 표시 폭 초과, 구조 토큰(`KK`, `<AA>`, `<FF>`, `<TT>`) 변형, 글꼴 코드북에 없는 글자, 일본어 잔존은 모두 0건입니다.
- 대사의 화자 이름 구간이 본문으로 덮어써지지 않았는지 12,582건 전수 감사했습니다. 손상 0건입니다.
- 도감 설명은 런타임 버퍼 규격인 최대 24열로 다시 배치했고, 줄 폭 초과·줄바꿈 표식 오류·끝 표식 누락 0건을 확인했습니다.
- 표본 220건을 별도 검수한 결과 오역·의미 왜곡(serious) 0건, 사소한 표현 문제 14건이었습니다.
- 빌드 산출물 독립 감사에서 텍스트 페이로드 62,506건 검증 실패 0건, 일본어 잔존 0건, 승인 외 실행 코드 변경 0건을 확인했습니다.
- 일본판 비대상 파일 277개는 원본과 동일합니다.
- xdelta 패치를 원본에 역적용해 결과 ISO의 SHA-256이 일치함을 확인했고, 배포되는 `apply_patch.ps1`을 실제로 실행해 재현을 검증했습니다.

## 재현성

이번 릴리스는 `tools/build_iso_from_maps.py`로 생성했습니다. 새 빌드를 만들기 전에 같은 도구로 v1.0.5를 그 번역 맵에서 다시 빌드해, 공개된 v1.0.5 ISO의 SHA-256(`7E9DA87B…`)과 바이트 단위로 일치하는 것을 먼저 확인했습니다. 따라서 v1.0.6과 v1.0.5의 차이는 번역 텍스트뿐입니다.

# 슈퍼로봇대전 GC 한글패치 v1.0.5

v1.0.4에서 정신기 설명 23건이 다른 정신기에 연결된 문제와 일부 축약·중복 표기를 수정한 릴리스입니다. 인물·로봇도감 런타임 수정, PDF 번역 품질 교정, 전투 유닛 그래픽 보호 방식과 일본판 이벤트 순서는 그대로 유지합니다.

## 정신기 표기 및 설명 수정

- 정신기 전체 명칭 30건, 축약 30건, 설명 29건을 일본판의 고정 ID와 일대일로 재검증
- 번뜩임 설명에 저격 효과가 표시되는 등 `r0003~r0025` 설명 23건이 연쇄 이동한 문제 수정
- 이미지처럼 표시되는 1글자 축약의 `자기→자`, `력→노`, `난→란` 오류 수정
- 실행 파일에 중복 저장된 `근성 도 근성→근성　초근성`, `카쿠란→교란`, `힘→기력`, `애→사랑` 표기 수정
- 보이는 설명 29건 전부를 최대 2줄·한 줄 23자·41바이트 이내로 정리하고 숨김 `？？？` 슬롯은 일본판 그대로 보존
- 원인이었던 정신기 설명 CSV 순번 매칭 결과를 안정 ID 교정으로 덮어쓰고 이름·축약·설명의 전체 배열을 검사하는 회귀 방지 도구 추가
- 정신기 관련 공통 UI 이미지 14건은 승인본과 픽셀 단위 일치, 필요한 한글 글리프 누락 0건 확인

## 인물·로봇도감 런타임 수정

- 인물도감 267건과 로봇도감 238건, 총 505건의 설명문을 일본판 도감 렌더러 규격에 맞춰 재배치
- 각 줄을 최대 24열·48바이트로 제한하고 ASCII 문자·숫자·공백을 2바이트 표시 셀로 정규화
- 줄바꿈 표식의 홀수 바이트 오프셋, 홀수 바이트 구간, 잘못된 16비트 문자 단위, 헤더 줄 수 불일치 및 끝 표식 누락 0건
- 문제 빌드 회귀 검사에서 2천 건 이상의 구조 오류를 검출하고, 수정 빌드에서는 동일 검사 오류 0건 확인
- 인물도감의 `기도・조타로` 오기를 `키도・죠타로`로 교정

## 번역 품질 교정

- 안정 ID 기준 실제 텍스트 2,778건 교정
- 초반 압사라스 실험장, 올리비 설득 장면 등 대사 95건 직접 교정
- `하야토`, `코우지`, `테츠야`, `아스카`, `진`, `츠토무`처럼 조사·호칭이 붙어 누락됐던 인명 표기 정리
- `모가미 중공`, `소울 세이버`, `다이오쟈`, `바쿠신거`, `수전기대`, `바쿠류오`, `사악수` 등 반복 용어 통일
- 다른 레코드 문장이 섞이거나 내부 placeholder만 제거됐던 도감 장문 13건을 일본어 원문에서 전면 재번역
- 번역 맵의 내부 placeholder와 일본어 문자 잔존 0건

## 시나리오 제목 이미지

- 시나리오 제목 39건을 PDF와 일본어 원문으로 재검토
- 이미 정확했던 2건을 제외한 독립 BMP 37개를 글꼴 제작에 사용한 굵은 맑은 고딕으로 다시 렌더링
- 에피소드 번호↔블록 번호 허용 목록과 원본 블록 SHA-256을 고정해 다른 UI 블록 오염 방지
- PDF 내부 표기가 충돌하는 경우 일본어 원문과 통용 표기를 우선하고, 남녀 주인공 이름은 각각 `아카츠키 아키미`와 `아카츠키 아케미`로 유지
